  3. class 표기법 통일

  4. 주관적인 해석이 들어갈 수 있는 클래스명 사용 지양

    .txt_bold ➡ .fw70061

    .divider_top ➡ .mt60188

    위 예시와 같이 단순 스타일 적용을 위한 class명은 속성명을 축약하여 명시적인 클래스명으로 일괄 변경하며,
    divider와 같이 주관적인 클래스명의 경우는 제거한다. ( 제거 요소에 대한 추가적인 검토 필요)

  5. 미디어쿼리에서의 float 초기화 필요성에 대한 고민221 ~ 236

    flex로 대체하되, 필요에 따라 사용

  6. .ib_wrap, .ib_item 필요성225

    콘텐츠 수평정렬에 쓰이는 class중 하나인(현재 .float_box / .ib_wrap 사용중) .ib_wrap의 경우 font-size:0 처리를 하므로 사용할 경우 스타일 지정이 필요.

    개인적인 의견으로는 inline-block으로 정렬이 필요한 경우가 있을 것으로 보아 class명만 수정하여 필요에 따라 사용하면 괜찮을듯 함.

    가능한 사용 지양, flex로 대체

  7. 효율적인 class명 활용에 대한 고민(예: position : absolute / compatibillity.css)249

    현황 : position값이 들어가는 모든 class에 position이 들어가있는 상태

    수정안 : 공통 속성이 들어가는 class명에 추가해서 쓰는 방식

    .pos.absolute { position: absolute;}
    .pos.relative { position: relative;}
    .pos.fixed { position:fixed}
    .pos.center { left: 50%; top: 50%; transform:translate(-50%,-50%)}

    위와 같이 코드를 작성했다고 가정했을 때 class명을 활용한다면? ➡ class="pos absolute center"

    이 방법을 사용하면 호환성 코드 또한 간결해질 것으로 판단됨

    위 예시와 같은 사용법으로 변경

  8. 미디어쿼리 관련

    1. 작성 순서

      현재 class 덩어리별로 작성되어있음
    2. 쿼리 기준에 대한 재정의

    1. 맨 하단으로 몰아서 작성.

    2. 해상도 사이즈

      media query 기준 현행가이드는 Wordpress 테마를 기준으로 작성되어있으나,
      가이드 작성 시점과 현재 시점을 비교해볼 때(가이드 작성 시점 : 2020년 / 현재 가이드 수정 시점 : 2022년)
      디바이스의 종류가 다양해지면서 자연스럽게 해상도도 다양해졌기에, 세분화해서 관리하는 것으로 한다.
      다만 기초적인 base 사이즈 지정하되, 갤럭시 z fold 시리즈와 같은 논외케이스는 프로젝트에서 필요할시 추가하여 작성하도록 한다.
      • /* common window device */

        @media screen and (max-width: 1920px) {}

      • /* common small laptop (13inch) */

        @media screen and (max-width: 1280px) {}

      • /* old laptop or 13inch laptop, iPad Pro series */

        @media screen and (max-width: 1024px) {}

      • /* small tablet - ipad mini */

        @media screen and (max-width: 768px){}

      • /* large mobile device */

        @media screen and (max-width: 425px){}

      • /* small mobile device - iphone mini series and old device */

        @media screen and (max-width: 360px){}
